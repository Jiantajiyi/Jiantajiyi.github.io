ftp服务器的搭建

搭建匿名访问的FTP服务器

服务端

  1. 装包
    [root@server_1 ~]# yum install vsftpd\* -y
  2. 关闭selinux
    [root@server_1 ~]# setenforce 0
  3. 防火墙放行ftp服务
    [root@server_1 ~]# firewall-cmd --add-service=ftp --per
    success
    [root@server_1 ~]# firewall-cmd --reload
    success
  4. 新建测试文件
    [root@server_1 ~]# echo this is ftp test! > /var/ftp/pub/test.txt
  5. 重启服务
    [root@server_1 ~]# systemctl restart vsftpd

客户端

装包

[root@server_2 ~]# yum install ftp\* -y

登录ftp

  1. 打开ftp客户端
    [root@server_2 ~]# ftp
    ftp> open 192.168.19.101

  2. 输入ftp用户名和空密码

下载测试文件

  1. 切换目录
    ftp> cd pub
    250 Directory successfully changed.
    ftp> ls

  2. 下载
    ftp> get test.txt

  3. 退出
    ftp> exit
    221 Goodbye.

查看测试文件

[root@server_2 ~]# cat test.txt
this is ftp test!

搭建配置系统用户的FTP服务器

服务端

  1. 关闭匿名访问
    [root@server_1 ~]# vim /etc/vsftpd/vsftpd.conf
    anonymous_enable=NO
  2. 创建要添加用户的家目录
    [root@server_1 ~]# mkdir -p /var/test1
  3. 添加用户指定为root组，指定家目录，不允许登录
    [root@server_1 ~]# useradd -g root -M -d /var/test1/ -s /sbin/nologin test1
  4. 更改权限
    [root@server_1 ~]# chown -R test1:root /var/test1/
  5. 给用户添加密码
    [root@server_1 ~]# echo redhat | passwd --stdin test1
    Changing password for user test1.
    passwd: all authentication tokens updated successfully.
  6. 新建测试文件
    [root@server_1 ~]# echo this is ftp test ! > /var/test1/test.txt
  7. 重启服务
    [root@server_1 ~]# systemctl restart vsftpd

客户端

  1. 登录ftp
    [root@server_2 ~]# ftp
    ftp> open 192.168.19.101

  2. 输入用户名test1和密码redhat

  3. 查看文件
    ftp> ls

  4. 下载文件
    ftp> get test.txt

  5. 退出
    ftp> exit

  6. 查看测试文件
    [root@server_2 ~]# cat test.txt
    this is ftp test !

搭建配置虚拟用户的FTP服务器

服务端

  1. 创建两个虚拟用户的家目录
    [root@server_1 ~]# mkdir -p /home/vsftpd/ftp1 /home/vsftpd/ftp2

  2. 加入两个用户admin和user，密码均为redhat
    [root@server_1 ~]# vim /etc/vsftpd/loginuser.txt

  3. 创建虚拟用户的数据库
    [root@server_1 ~]# db_load -T -t hash -f /etc/vsftpd/loginuser.txt /etc/vsftpd/login.db

  4. 设置数据库访问权限
    [root@server_1 ~]# chmod 600 /etc/vsftpd/login.db

  5. 修改配置文件，允许PAM验证
    [root@server_1 ~]# vim /etc/pam.d/vsftpd

  6. 新建系统用户vsftpd，不允许登录，指定家目录
    [root@server_1 ~]# useradd -d /home/vsftpd/ -s /bin/false vsftpd

  7. 更改权限
    [root@server_1 ~]# chown -R vsftpd:vsftpd /home/vsftpd/

  8. 修改vsftpd.conf
    [root@server_1 ~]# grep -v '^#' /etc/vsftpd/vsftpd.conf

  9. 创建用户配置文件夹user_conf
    [root@server_1 ~]# mkdir /etc/vsftpd/user_conf

  10. 为每个用户创建配置文件
    [root@server_1 ~]# cd /etc/vsftpd/user_conf/
    [root@server_1 user_conf]# touch admin user

  11. 编辑admin的配置文件，读写权限
    [root@server_1 user_conf]# vim admin

  12. 将admin添加到允许用户列表中去
    [root@server_1 user_conf]# vim /etc/vsftpd/chroot_list
    admin

  13. 编辑user的配置文件，只读权限
    [root@server_1 user_conf]# vim user

  14. 给ftp1目录所有权限,ftp2目录去除写权限
    [root@server_1 user_conf]# chmod 777 /home/vsftpd/ftp1
    [root@server_1 user_conf]# chmod a-w /home/vsftpd/ftp2

  15. 新建测试文件
    [root@server_1 vsftpd]# echo this is test > /home/vsftpd/ftp1/test.txt
    [root@server_1 vsftpd]# echo this is ftp2 test > /home/vsftpd/ftp2/test.txt
  16. 重启服务
    [root@server_1 user_conf]# systemctl restart vsftpd

客户端

以admin用户登录，有读写权限和切换到上级目录的权限

  1. 登录ftp
    [root@server_2 ~]# ftp
    ftp> open 192.168.19.101

  2. 输入用户名admin和密码redhat

  3. 查看
    ftp> ls

  4. 下载测试文件
    ftp> get test.txt

  5. 尝试删除文件
    ftp> delete test.txt
    250 Delete operation successful.

  6. 尝试创建文件夹
    ftp> mkdir test
    257 “/home/vsftpd/ftp1/test” created

  7. 切换到上一目录
    ftp> cd ..
    250 Directory successfully changed.
    ftp> ls

  8. 退出
    ftp> exit
    221 Goodbye.

  9. 查看
    [root@server_2 ~]# cat test.txt
    this is test

以user用户登录，只有读的权限，没有切换到上级目录的权限

  1. 切换目录成功但只能显示当前目录下的测试文件
    ftp> cd /
    250 Directory successfully changed.
    ftp> ls

  2. 下载测试文件
    ftp> get test.txt

  3. 尝试删除文件但错误
    ftp> delete test.txt
    550 Delete operation failed.

  4. 尝试创建文件夹但错误
    ftp> mkdir test
    550 Create directory operation failed.
  5. 退出
    ftp> exit
    221 Goodbye.
  6. 查看测试文件
    [root@server_2 ~]# cat test.txt
    this is ftp2 test
